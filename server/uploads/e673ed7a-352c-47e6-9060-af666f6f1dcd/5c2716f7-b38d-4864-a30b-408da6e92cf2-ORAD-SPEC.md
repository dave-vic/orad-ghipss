## 3. Database Schema (Prisma)

```prisma
// server/prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

enum Role {
  admin
  member
  viewer
}

enum Action {
  login
  login_fail
  logout
  view
  download
  upload
  delete
  user_create
  role_change
  deactivate
}

model User {
  id         String    @id @default(uuid())
  name       String
  email      String    @unique
  username   String    @unique
  password   String                        // bcrypt hash
  role       Role      @default(viewer)
  active     Boolean   @default(true)
  createdAt  DateTime  @default(now())
  lastLogin  DateTime?

  documents  Document[]
  logs       ActivityLog[]

  @@map("users")
}

model Folder {
  id           String    @id @default(uuid())
  name         String
  description  String?
  icon         String    @default("рҹ“Ғ")
  allowedRoles Role[]
  createdAt    DateTime  @default(now())

  documents    Document[]

  @@map("folders")
}

model Document {
  id          String    @id @default(uuid())
  folderId    String
  folder      Folder    @relation(fields: [folderId], references: [id])
  name        String
  storageKey  String                        // S3 object key
  sizeBytes   BigInt
  mimeType    String
  uploadedBy  String
  uploader    User      @relation(fields: [uploadedBy], references: [id])
  uploadedAt  DateTime  @default(now())
  deletedAt   DateTime?                     // soft delete

  @@map("documents")
}

model ActivityLog {
  id          String   @id @default(uuid())
  userId      String
  user        User     @relation(fields: [userId], references: [id])
  action      Action
  targetType  String?                       // "document" | "folder" | "user"
  targetId    String?
  targetName  String?                       // snapshot of name at event time
  ipAddress   String?
  userAgent   String?
  createdAt   DateTime @default(now())

  @@map("activity_logs")
}
```

## 5. API Reference

### 5.1 Auth

| Method | Route | Auth | Body / Params | Response |
|---|---|---|---|---|
| POST | `/api/auth/login` | None | `{ username, password }` | Sets JWT cookie В· returns `{ user }` |
| POST | `/api/auth/logout` | Session | вҖ” | Clears cookie |
| GET | `/api/auth/me` | Session | вҖ” | `{ id, name, role, email }` |

**Login flow:**
1. Find user by `username`
2. Compare password with `bcrypt.compare`
3. If `user.active === false` вҶ’ 403
4. Sign JWT: `{ sub: user.id, role: user.role }`, expiry 8h
5. Set cookie: `httpOnly: true, sameSite: 'strict', secure: true`
6. Log `action: 'login'` to ActivityLog
7. Update `user.lastLogin`

### 5.2 Folders

| Method | Route | Auth | Description |
|---|---|---|---|
| GET | `/api/folders` | Any | Returns folders where `role IN allowedRoles` |
| POST | `/api/folders` | Admin | Create folder `{ name, description, icon, allowedRoles }` |
| PATCH | `/api/folders/:id` | Admin | Update folder metadata |
| DELETE | `/api/folders/:id` | Admin | Soft-delete (set `deletedAt`) |

### 5.3 Documents

| Method | Route | Auth | Description |
|---|---|---|---|
| GET | `/api/folders/:folderId/documents` | Any (role-checked) | List non-deleted docs in folder |
| POST | `/api/folders/:folderId/documents` | Admin | Upload file via multipart вҶ’ S3 |
| GET | `/api/documents/:id/download` | Any (role-checked) | Returns pre-signed S3 URL (5min expiry) В· logs `download` |
| DELETE | `/api/documents/:id` | Admin | Soft-delete |

**Upload flow:**
1. Multer receives file in memory
2. Generate S3 key: `${folderId}/${uuid()}-${originalname}`
3. `PutObjectCommand` with `ContentType` and `ServerSideEncryption: 'AES256'`
4. Save Document record with `storageKey`, `sizeBytes`, `mimeType`
5. Log `action: 'upload'`

**Download flow:**
1. Fetch Document, verify `deletedAt === null`
2. Check user role against folder's `allowedRoles`
3. `GetObjectCommand` вҶ’ `getSignedUrl` with `expiresIn: 300`
4. Log `action: 'download'`
5. Return `{ url, filename, expiresIn: 300 }`

### 5.4 Users (Admin only)

| Method | Route | Body | Description |
|---|---|---|---|
| GET | `/api/users` | вҖ” | All users, sorted by `createdAt` desc |
| POST | `/api/users` | `{ name, email, username, password, role }` | Create user (bcrypt password) В· log `user_create` |
| PATCH | `/api/users/:id/role` | `{ role }` | Change role В· log `role_change` |
| PATCH | `/api/users/:id/status` | `{ active }` | Toggle active В· log `deactivate` if false |
| DELETE | `/api/users/:id` | вҖ” | Hard delete (confirm user is not self) |

### 5.5 Activity Logs

| Method | Route | Auth | Query params | Description |
|---|---|---|---|---|
| GET | `/api/logs` | Admin | `user`, `action`, `from`, `to`, `page`, `limit` | Paginated full log |
| GET | `/api/logs/me` | Any | `action`, `from`, `to` | Current user's log only |
| GET | `/api/logs/export` | Admin | Same as `/api/logs` | Returns CSV file download |

## 6. Middleware Patterns

### authenticate.js
```javascript
export const authenticate = asyncHandler(async (req, res, next) => {
  const token = req.cookies?.token;
  if (!token) return res.status(401).json({ error: 'Not authenticated' });

  const payload = jwt.verify(token, process.env.JWT_SECRET);
  const user = await prisma.user.findUnique({ where: { id: payload.sub } });

  if (!user || !user.active) return res.status(403).json({ error: 'Access denied' });

  req.user = user;
  next();
});
```

### requireRole.js
```javascript
export const requireRole = (...roles) => (req, res, next) => {
  if (!roles.includes(req.user.role)) {
    return res.status(403).json({ error: 'Insufficient permissions' });
  }
  next();
};

// Usage:
router.post('/users', authenticate, requireRole('admin'), createUser);
```

### Folder access check
```javascript
// Reusable function used in folder + document controllers
export const assertFolderAccess = (folder, userRole) => {
  if (!folder.allowedRoles.includes(userRole)) {
    throw new ForbiddenError('You do not have access to this folder');
  }
};
```

## 8. Security Checklist

### Passwords
- [ ] `bcrypt.hash(password, 12)` on create and password reset
- [ ] Never log, return, or store plain-text passwords
- [ ] Never return `password` field in any API response (use Prisma `select` to exclude)

### JWT & Sessions
- [ ] Cookie flags: `httpOnly: true`, `secure: true`, `sameSite: 'strict'`
- [ ] JWT expiry: `8h`
- [ ] On password change or deactivation: clear the user's cookie immediately
- [ ] Sign with `process.env.JWT_SECRET` (min 32 chars, random)

### File Storage
- [ ] S3 bucket ACL: **private** вҖ” no public access
- [ ] Pre-signed URLs expire in **300 seconds**
- [ ] Server-side encryption: `AES256`
- [ ] Never expose `storageKey` (S3 object key) in API responses

### API Security
- [ ] Rate limit login: 10 attempts / minute per IP вҶ’ 429
- [ ] CORS: whitelist frontend origin only (`process.env.CLIENT_URL`)
- [ ] Helmet.js for security headers (CSP, HSTS, X-Frame-Options, etc.)
- [ ] Validate all request bodies with Zod before touching the DB
- [ ] Never trust `role` from the request body when creating sessions

### Activity Logs
- [ ] No `UPDATE` or `DELETE` route on `activity_logs` вҖ” write-only via `log.service.js`
- [ ] Log `login_fail` events (track username attempted, IP вҖ” no password)
- [ ] Admin cannot delete their own account

## 13. Definitions

| Term | Meaning |
|---|---|
| RBAC | Role-Based Access Control вҖ” permissions set at role level, not per user |
| JWT | JSON Web Token вҖ” encrypted credential stored in the browser cookie |
| HTTP-only cookie | A cookie JavaScript cannot read вҖ” protects against XSS token theft |
| Pre-signed URL | A temporary, time-limited S3 download link вҖ” expires after 5 minutes |
| Soft delete | Marking a record as deleted without removing it from the database |
| Bcrypt | Password hashing algorithm вҖ” stores a one-way scrambled version of the password |
| Middleware | Code that runs between a request arriving and the controller handling it |
| Seed data | Initial data inserted into the database when it is first set up |
